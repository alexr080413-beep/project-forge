# Forge Roadmap

Every Event. Every Inject. Every Exercise.

Human Command. Machine Assistance.

Forge is an open-source digital operations platform for modern Exercise Control. People create training. Forge manages information.

## Forge Vision

Forge exists to connect the full exercise lifecycle into one controlled operational workflow. It helps Exercise Control teams manage planning, execution, intelligence, injects, review, products, assessment, audit records, and exercise archives without losing human authority or scenario fidelity.

Forge is not an autonomous exercise controller. It is a modular information platform that prepares, validates, routes, records, and surfaces the work controllers need to run modern exercises.

The current milestone is Forge Studio v0.1: the first runnable local demonstration of the Forge operating model.

## Forge 1.0 Goals

Forge 1.0 should represent a professional enterprise platform foundation for Exercise Control.

Goals:

- Stable Forge Core service boundaries.
- A usable Forge Studio application for multi-organization, multi-exercise workflows.
- Exercise lifecycle management across planning, preparation, execution, assessment, and archive.
- Human-controlled inject and product review workflow.
- Product plugin framework.
- Mission Replay foundation.
- Operational analytics foundation.
- Security and role model suitable for enterprise growth.
- Deterministic local development and test behavior.
- Clear documentation for users, contributors, maintainers, and extension authors.

Forge 1.0 should remain honest about boundaries. It does not need live external integrations, production identity, cloud deployment, or AI provider execution to be valuable.

## Long-Term Direction

Long-term, Forge should become the operating system for modern Exercise Control.

The platform direction includes:

- Forge Studio as the main command-center application.
- Forge Core as the deterministic service layer.
- Forge API for integration and automation.
- Forge SDK for plugins, profiles, products, and workflows.
- Forge CLI for local operations and developer workflows.
- Forge Replay for mission reconstruction and after-action analysis.
- Forge Assist for bounded AI-supported controller work.
- Forge Mobile for lightweight review and situational awareness.
- Cross-exercise analytics and reusable exercise archives.
- Coalition-aware profiles, mappings, permissions, and release workflows.

The guiding rule remains constant: Forge assists; people decide.
