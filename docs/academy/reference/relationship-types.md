# Relationship Types

Relationships are graph edges between operational assets.

| Relationship | Meaning |
| --- | --- |
| `supports` | Supports an objective or outcome. |
| `produces` | Produces a product, finding, or artifact. |
| `triggers` | Initiates another event or asset. |
| `depends_on` | Requires another asset or condition. |
| `assigned_to` | Assigned to a controller or role. |
| `observes` | Records evidence about another asset. |
| `evaluates` | Evaluates behavior, evidence, or objective performance. |
| `references` | Cites or traces to another asset. |
| `related_to` | General relationship. |
| `precedes` | Occurs before another asset. |
| `follows` | Occurs after another asset. |
| `contains` | Owns, scopes, or contains another asset. |
| `inherits` | Uses a template or inherited structure. |
