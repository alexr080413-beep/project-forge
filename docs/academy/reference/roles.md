# Roles

Roles describe common Forge responsibilities.

| Role | Primary Responsibility |
| --- | --- |
| Exercise Director | Owns exercise authority, readiness, approval posture, and archive decisions. |
| Exercise Controller | Manages injects, timeline actions, and controller coordination. |
| Intelligence Chief | Maintains intelligence picture and operational traceability. |
| Reviewer | Approves, rejects, or returns items for revision. |
| System Administrator | Supports setup, startup, configuration, and access assumptions. |
| Developer | Maintains code, models, APIs, plugins, architecture, and documentation. |

## Current Boundary

Forge Studio MVP does not implement real authentication. Human review authority is modeled through workflow and documentation.
