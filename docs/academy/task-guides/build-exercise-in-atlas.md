# Build Exercise In Atlas

Use Atlas to draft the exercise structure before execution.

## Steps

1. Open `Exercise Designer`.
2. Confirm the selected organization and exercise in the global header.
3. Review the Object Library for planning object categories.
4. Add objectives with priority, success criteria, and linked assets.
5. Add or edit controller assignments, responsibilities, linked objectives, and linked injects.
6. Create injects with controller, objective, priority, notes, and planned time.
7. Add, edit, delete, or move timeline events.
8. Select a planned item in the canvas.
9. Review linked objectives, related injects, assigned controller, produced products, follow-on events, and validation warnings.
10. Use `Save Draft` to record the draft state in the local audit trail.
11. Use `Validate` to refresh readiness checks.
12. Use `Publish` only as a human-gated publish request. It does not release the plan.

## Alpha Boundary

Atlas Alpha is interactive and updates the Exercise Data Engine, Mission Control counts, Knowledge Graph relationships, validation status, activity feed, and audit log during the running session.

Durable persistence, drag and drop, real publishing, collaborative editing, and versioned exercise plans are not implemented yet.
