# Development Policy

This policy applies to future Forge development prompts, issues, tickets, and pull requests.

## Required Future Codex Prompt Sections

Every future Codex implementation prompt must include:

- Objective.
- Operational Value.
- Requirements.
- Testing.
- Documentation Updates.
- Forge Academy Updates.
- Reference Exercise Updates.
- Definition of Done.
- Release Notes.
- Architecture Updates.

## Documentation Rule

Features are not complete until relevant documentation and training material are updated.

## Project Sentinel Rule

Project Sentinel is the canonical Forge reference implementation.

Future work must state whether Project Sentinel validation is required. If required, the change should update Sentinel examples or explicitly verify that Sentinel still covers the workflow.

## Architecture Rule

Architecture-changing work must update architecture documentation and create or update an ADR when the decision has long-term consequences.

Major proposals should link an RFC.

## Release Rule

Release-relevant work must update changelog or release notes.

## UI Rule

UI changes must include screenshot or browser verification when feasible and must follow Forge Studio design system guidance.
