# Support

Forge support starts with documentation and clear issue reports.

## Bug Reports

Use the GitHub bug report template. Include steps to reproduce, expected behavior, actual behavior, environment details, and safe logs.

## Questions

Use project discussions or issue threads when available. Include the role, workflow, and documentation page you were using.

## Documentation Requests

Use the documentation issue template for unclear, missing, or outdated docs.

## Roadmap

Roadmap discussion should connect proposals to exercise control value, human review, auditability, and documentation impact.

## Community

Forge community participation should stay professional, operational, and focused on helping users conduct better exercises.
