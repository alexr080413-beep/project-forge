# Icons

Forge icons should support command-center clarity rather than decoration.

## Style

| Attribute | Direction |
| --- | --- |
| Icon family | Outline icons |
| Stroke width | 2px |
| Corners | Rounded edges |
| Primary colors | Forge Orange `#F6A86E` or Ash `#C7CDD6` |
| Filled icons | Avoid unless required for contrast or critical status visibility |
| Tone | Operational, simple, legible |

## Recommended Icon Concepts

| Concept | Use |
| --- | --- |
| Shield | Security, authorization, control boundaries. |
| Timeline | Exercise timeline, phase, schedule, sequence. |
| Clipboard check | Review, approval, QA completion. |
| Alert triangle | Warning, critical finding, escalation. |
| File text | Product, draft, report, output. |
| Search | Knowledge lookup, entity lookup, archive search. |
| Activity | Metrics, health, pipeline activity. |
| Users | Controllers, review teams, workspace participants. |
| Database | Storage, archive, knowledge base. |
| Map | Exercise map, location, terrain, common operational picture. |

## Rules

- Pair unfamiliar icons with visible labels or tooltips.
- Do not use cartoon icons.
- Do not use emoji as interface icons.
- Do not mix heavy filled icons with outline icons in the same control group.
- Do not use weapons-focused icons as default operational symbols.
- Use status labels with icons so color is never the only state indicator.

## Sizes

| Context | Size |
| --- | --- |
| Dense tables | 16px |
| Buttons and controls | 16px to 18px |
| Sidebar navigation | 18px to 20px |
| Cards and dashboard widgets | 20px |
| Empty states | 24px |

## Color Rules

- Use Ash for neutral navigation and secondary controls.
- Use Forge Orange for selected navigation, active controls, and primary actions.
- Use semantic status colors only when a state is being communicated.
- Avoid multi-color icons unless the icon is a product illustration, not an interface control.
